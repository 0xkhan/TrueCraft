Initial pass on server-side lighting

Published Jun 27, 2015

I've started building out support for block and sky light on TrueCraft servers. It may seem odd for the server to care about lighting, but it's necessary for things like determining where to spawn mobs or whether or not crops can grow. You can see in the screenshot here - the base of the tree is shaded by its leaves, a first for TrueCraft.

There are still some problems that I plan on sorting out next time I sit down to work on TrueCraft. Most importantly, block light is untested and the performance is pretty bad (making it unplayable as you start to make changes to the world and queueing up lighting recalculations). These issues will be solved soon.

Additionally, mrpimpunicorn has started moving us away from MonoGame and towards OpenTK. I've frozen client development until this move is complete. The impetus for the move is that MonoGame has really bad support for shaders right now, and combined with some other platform-specific problems it's clearly not a good choice moving forward. The bonus is that with OpenTK we'll probably have a much easier time maintaining good support for Windows and OS X in addition to Linux.

Once the client move is complete, I hope that server-side lighting will be mature and I can spend some time adding better lighting to the client as well.

Thanks a ton to dexter0 who was kind enough to reverse engineer Minecraft's lighting subsystem and provide clean room documentation for me to base my work on. Remember, if you're a developer who has read the decompiled Minecraft source, you aren't out of options for helping the project out!

I also made it so that you need to use certain tools to mine certain blocks, like stone or cobweb, but that's not interesting so I won't mention it here.
